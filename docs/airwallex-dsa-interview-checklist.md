# Airwallex DSA Interview Checklist

## Before Coding

- Restate problem and examples.
- Ask about input size.
- Ask about duplicates, nulls, empty input, ordering, and mutation.
- State chosen pattern.

## While Coding

- Name variables by role.
- Keep helpers small.
- Test happy path, edge path, and tricky path.

## After Coding

- Explain time complexity.
- Explain space complexity.
- Mention one fintech production follow-up: idempotency, ordering, duplicate IDs, precision, or concurrency.
